The testcase provided along with the problem is also for practice.

  ①   Test takers who will be using C or C++ to write their answer,

  ㅇ   please select C++ as for the Language before submitting the answer.

  ②   Consists of the Main part and User Code part.

  ㅇ   A.  Main Part         : Cannot be edited. But note that a validation logic can be added in here

  ㅇ   A.  Main Part         : for the purposes of solution evaluation such as to detect an abnormal solution.

  ㅇ   B.  User Code Part  : Code which the test taker must actually write.

  ㅇ   B.  User Code Part  : Make sure that standard input/output functions are NOT included when submitting the code.

  ③   Cautions When Programming on Local PC

  ㅇ   A.  2 files must be generated. ( main.cpp / solution.cpp or Solution.java / UserSolution.java or main.py / solution.py )

  ㅇ   B.  Please copy the Main part into main.cpp or Solution.java.

  ㅇ   C.  To use the sample_input.txt, uncomment the code (commented)

  ㅇ   C.  which uses the file as the standard input within the Main part code.

  ㅇ   D.  Delete or comment all standard input/output functions for debugging before submitting your code.

[Problem Description]

There is one grid-shaped glacier on the ice planet.

The glacier consists of regions whose sizes are 1 x 1.

The size of the glacier is R x C and it is surrounded by sea.

[Fig. 1] is when R and C are both 6.

The coordinates of each region in the glacier are expressed as (Row number, Column number). The numbers start from 0.

The row number is until R-1. The column number is until C-1.

[Fig. 1]

 

You want to explore a certain region in the glacier.

The region that has to be explored is called exploration zone.

As humans cannot go to the ice planet, an exploration robot is sent instead.

The region where the robot lands is called drop zone.

As soon as the robot lands on the planet, it stops moving in the drop zone.

Given that it is extremely difficult to drop the robot on the ice planet, it is difficult to exactly drop it in the exploration zone.

In other words, the drop zone and the exploration zone may differ.

[Fig. 2] is when the coordinates of the drop zone are (2, 1), while the coordinates of the exploration zone are (4, 4).

[Fig. 2]

 

The robot can only move in four directions—up, down, left, and right.

As the top of the glacier is slippery, once the robot moves there, it cannot change its direction nor stop moving until it hits a place that is not a glacier region.

When the robot hits the sea, it will fall into it.

For instance, when moving the robot to whichever direction in the drop zone of [Fig. 2], it will fall into the sea without arriving in the exploration zone. (Fig. 3)

Once the robot falls into the sea, it will be broken and cannot move anymore.

[Fig. 3]

 

The robot’s scope of exploration is too narrow.

Thus, it can explore only when it is in the exploration zone.

Instead, the speed of exploration is extremely fast.

When the robot is in the exploration zone for even just a moment, the exploration is completed.

In other words, the exploration is completed even when the robot just passes the exploration zone.

Given that the coordinates of the drop zone and the exploration zone are (4, 1) and (4, 4), respectively, if the robot moves to the right, it will pass the exploration zone, thereby completing the exploration. (Fig. 4)

After this, you do not have to worry about the robot falling into the sea.

[Fig. 4]

 

For seamless exploration, structures whose sizes are 1 x 1 were already installed in the glacier.

Each structure is installed in a way that it fits into one region.

The installed structure does NOT move and disappear.

The structure-installed region is no longer a glacier region.

The robot cannot exist in a region where a structure is installed. Thus, it cannot pass the region.

When the robot hits a structure, it stops in the glacier region right before the structure-installed region.

When the robot hits a structure, the damage inflicted upon the structure is 1.

Given that the coordinates of the robot and the structure-installed region are (4, 1) and (0, 1), respectively, when the robot moves up, the damage to the structure is 1 and the robot stops at (1, 1). (Fig. 5)

[Fig. 5]

 

Given that the coordinates of drop zone and exploration zone are (2, 1) and (4, 4), respectively, and the coordinates of regions where structures are installed are (0, 1) and (1, 5), when the robot moves to the up, right and down in order, the exploration is completed. (Fig. 6)

Then, the sum of the damage to the structure is 2.

[Fig. 6]

 

Implement each required function by referring to the following API description.

※ The function signature below is based on C/C++. As for other languages, refer to the provided Main and User Code.

 

The following is the description of API to be written in the User Code.

void init(int R, int C, int M, int mStructureR[], int mStructureC[])

This function is called in the beginning of each test case.

 

The size of the glacier is R x C.

 

The number of structures installed is M.

The coordinates of the region where the ith structure is installed are (mStructureR[i], mStructureC[i]). (0 ≤ i ≤ M-1)

It is guaranteed that the structures are all installed in different regions.

 

Parameters

   R : The number of rows (5 ≤ R ≤ 700)

   C : The number of columns (5 ≤ C ≤ 10,000)

   M : The number of structures that is installed (0 ≤ M ≤ 30,000)

   mStructureR[] : The row number of region where a structure is installed (0 ≤ mStructureR[] ≤ R-1)

   mStructureC[] : The column number of region where a structure is installed (0 ≤ mStructureC[] ≤ C-1)

int minDamage(int mStartR, int mStartC, int mEndR, int mEndC)

This function returns the minimum total sum when the exploration is completed in a way that the total sum of damages to the structures is minimum.

If the exploration cannot be completed, return -1.

 

The coordinates of drop zone are (mStartR, mStartC).

The coordinates of exploration zone are (mEndR, mEndC).

 

It is guaranteed that the drop zone and the exploration zone are NOT where a structure is installed.

 

Parameters

   mStartR : The row number of drop zone (0 ≤ mStartR ≤ R-1)

   mStartC : The column number of drop zone (0 ≤ mStartC ≤ C-1)

   mEndR : The row number of exploration zone (0 ≤ mEndR ≤ R-1)

   mEndC : The column number of exploration zone (0 ≤ mEndC ≤ C-1)

 

Return

   The minimum total sum of the damages inflicted upon structures
